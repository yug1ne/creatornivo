INPUT HANDLING

Use {{keyPageFacts}} and the other supplied fields as the only approved factual basis. Never invent features, prices, dates, locations, credentials, statistics, awards, reviews, testimonials, results, guarantees, partnerships, certifications, or regulated claims. Do not expose private data or credentials. For high-stakes pages, use cautious non-advisory wording and no outcome guarantees.

When an optional field is blank, infer only neutral editorial choices. Omit elements that require missing facts or URLs. Do not write “not provided,” create fake URLs, or leave empty customer-facing placeholders. Add a short verification note only when missing information blocks correct implementation.

SEARCH AND INTENT

* Use {{searchIntent}}. If it is Auto or blank, infer intent from the page type, goal, audience, topic, and keyword.
* Use {{primaryKeyword}} naturally in the title when clarity allows and in the description or a close variant when useful.
* Use {{secondaryKeywords}} selectively; never force every term.
* Apply {{geoTarget}} only when the page genuinely serves that location.
* Follow {{termsGuidance}} for required, preferred, or excluded wording.
* Match {{pageType}}: identify the offer on product or service pages, the information value on articles, the location on local pages, and the overall proposition on homepages.
* Do not promise content, features, outcomes, downloads, or services absent from the page.
* Avoid keyword stuffing, repeated boilerplate, vague titles, excessive punctuation, and sensational wording.
* Never promise rankings, traffic, indexing, click-through rates, or a guaranteed search appearance.

SEO TITLE AND DESCRIPTION

Create {{variantCount}} distinct, paired title and description options.

Titles:

* Descriptive, concise, page-specific, and consistent with the visible page topic.
* Safe editorial target: roughly 45–60 characters when the language permits, not an official maximum.
* Prioritize clarity over mechanically reaching a number.

Apply {{titleBranding}}:

* Auto: use {{brandName}} only when useful and space allows.
* Include brand: add {{brandName}} once, usually at the end with a simple separator.
* Exclude brand: omit it unless that would make the page misleading.

Descriptions:

* Summarize the specific page and its useful differentiator.
* Safe editorial target: roughly 135–155 characters when practical, not an official maximum.
* Use one proportionate action phrase only when it suits {{primaryGoal}}.
* Avoid fake urgency, fake scarcity, unverifiable superlatives, and guaranteed outcomes.
* Make each variant meaningfully different, not a synonym swap.

SLUG, CANONICAL, AND INDEXING

Recommend one primary slug or path.

Follow {{slugPreference}}:

* Auto: short, descriptive, lowercase, and hyphen-separated.
* Shortest clear slug: remove unnecessary modifiers.
* Keyword-led slug: use the primary keyword naturally.
* Improve existing slug: revise {{currentSlug}} only when useful.
* Keep existing slug: preserve {{currentSlug}} and review it.

Prefer about 3–6 meaningful words when possible. Do not include a domain, tracking parameters, unsupported dates, or invented folders. If a live slug changes, note that a redirect may be needed without claiming one exists.

For {{indexingPreference}}:

* Auto: omit an explicit robots tag unless clearly necessary.
* Index, follow: note that it is normally the default and omit a redundant tag.
* Noindex, follow: output that directive.
* Noindex, nofollow: output it and flag its restrictive effect.

Use {{canonicalUrl}} when supplied. Otherwise use {{pageUrl}} only if it is an absolute, clean intended canonical URL. If neither is usable, omit the canonical tag and add a verification note. Never fabricate a URL.

SOCIAL METADATA

Only include this section when {{includeSocialMetadata}} is enabled. Follow {{socialPlatforms}}.

Create faithful social titles and descriptions; they may be slightly more conversational. Use safe editorial targets of about 40–65 characters for titles and 80–120 characters for descriptions when practical.

Use {{socialImageUrl}} only when it is a supplied absolute URL. Use {{socialImageAlt}} only when provided, refining wording without inventing visual details.

For Open Graph, use only values supported by inputs: `og:title`, `og:description`, `og:type`, `og:url`, `og:image`, and `og:image:alt`. Use `article` for article-like pages and `website` for general pages unless another type is clearly justified.

For X Cards, use `summary_large_image` only when an image URL is supplied; otherwise use `summary`. Do not invent handles, image dimensions, creators, dates, or app IDs. Omit incomplete tags and list only genuinely missing implementation values in verification notes.

HTML RULES

Escape special characters correctly in HTML attributes. Do not include meta keywords, structured data, hreflang, viewport, charset, analytics, tracking, verification tags, or unrelated markup.
