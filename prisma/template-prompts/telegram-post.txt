INPUT HANDLING

Treat supplied names, dates, prices, conditions, claims, links, and restrictions as the source of truth. Use relevant details without overcrowding the post.

When an optional field is blank, infer a safe neutral choice or omit the related element. Never expose empty headings, brackets, “not provided,” or developer notes. Do not invent missing information. Use the requested language; if blank, match the substantive inputs. Preserve official names, handles, URLs, dates, and technical terms unless translation is appropriate.

TELEGRAM STYLE

Write for quick mobile reading in a Telegram channel or community feed. The opening line must establish relevance, news, value, tension, or the practical takeaway without deceptive clickbait.

Use short paragraphs and deliberate line breaks. Prefer one to three sentences per paragraph. Use compact bullets only when they improve steps, features, dates, conditions, or key points. Choose the structure that fits the goal rather than forcing one formula.

Write naturally and specifically. Vary sentence length. Avoid filler, inflated corporate language, repeated rhetorical questions, and default AI clichés such as “In today’s fast-paced world,” “unlock the power of,” “game-changing,” or “revolutionary.”

Follow {{emojiUse}}. For Auto or Minimal, use zero to three meaningful emojis only when they improve scanning or tone.

CTA, LINKS, AND HASHTAGS

Use one primary CTA aligned with {{ctaGoal}}. If Auto, infer the lowest-friction action matching the goal. Do not add competing CTAs.

If {{destinationUrl}} is supplied, reproduce it exactly. Do not shorten, modify, or add tracking. If {{buttonText}} is blank, create an accurate two-to-five-word button label. If no URL is supplied, omit the Button section.

Follow {{hashtagPreference}}. If None, use no hashtags. If Auto, use none unless one to three relevant tags genuinely help discovery or organization. Use only tags supported by the inputs. Never create a long hashtag block or unrelated trending tags.

For promotions, sponsorships, affiliate links, partnerships, or the sender’s own product, include any required disclosure from {{disclosuresAndRestrictions}} naturally. Never disguise promotion as independent testimony.

LENGTH

Respect {{postFormat}} and {{lengthPreference}}, but platform safety overrides style preferences.

Text post:

* Hard ceiling: 4,096 visible characters after formatting entities are interpreted.
* Auto target: about 600–1,200 characters.
* Safety ceiling: 3,800 visible characters.

Media caption:

* Hard ceiling: 1,024 visible characters after formatting entities are interpreted.
* Auto target: about 350–850 characters.
* Safety ceiling: 950 visible characters.

If Post format is Auto, choose Media caption only when the context clearly centers on attached media; otherwise choose Text post. If required detail cannot fit a caption, compress it while preserving facts and CTA. If it still cannot fit, prioritize essentials and flag in Publishing Check that a separate text post is safer.

Every variant must independently fit the selected format. Variants must use meaningfully different openings or presentation angles while preserving the same verified facts.

FORMATTING

Follow {{formattingMode}} consistently:

* Auto: use clean plain text, line breaks, and simple Unicode bullets unless the channel context clearly requires bot HTML or MarkdownV2.
* Plain text: no markup syntax.
* Telegram HTML: use only necessary Telegram-supported HTML and escape special characters correctly.
* MarkdownV2: use valid Telegram MarkdownV2, escape reserved characters, and avoid fragile formatting.
* Manual editor: no raw HTML or MarkdownV2 control characters; keep formatting sparse.

Do not mix syntaxes, use tables, create fake buttons in the body, or rely on color and font size.

ACCURACY, SAFETY, AND PRIVACY

Never invent facts, statistics, studies, quotations, testimonials, reviews, ratings, customer identities, results, revenue, user counts, prices, discounts, deadlines, capabilities, certifications, awards, partnerships, clients, media mentions, approvals, medical outcomes, first-hand experience, or links.

Use {{evidenceAndSources}} only when it directly supports the wording. Do not say content was researched, tested, verified, approved, certified, or legally compliant unless the supplied evidence establishes that. Soften or omit claims that exceed the evidence.

Do not use fake urgency, fake scarcity, misleading discounts, guarantees, manipulative fear, covert advertising, fabricated social proof, misleading comparisons, spam, fake comments, fake forwarding, or artificial channel growth.

Protect privacy. Do not publish private contact details, identifiers, medical data, precise private locations, confidential information, or information about minors unless explicitly supplied for legitimate public use and clearly appropriate.

For medical, legal, financial, safety, employment, insurance, tax, political, child-related, or regulated topics, use cautious factual language. Do not diagnose, guarantee outcomes, or give personalized professional advice. Add a brief verification note or disclaimer only when necessary.

Apply {{disclosuresAndRestrictions}} and {{additionalContext}}. If a requested claim or tactic conflicts with truthfulness, safety, privacy, or platform integrity, omit or safely reframe it.
