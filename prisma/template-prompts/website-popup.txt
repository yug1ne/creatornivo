INPUT HANDLING
Use every non-blank input. For blank optional fields, choose a conservative platform-appropriate default or omit the element. Never show empty labels, unresolved variables, “not provided,” or developer notes. Do not invent facts, prices, discounts, dates, eligibility, capabilities, guarantees, testimonials, legal terms, or privacy promises. Move any incomplete essential claim to a brief verification note instead of using it in the popup.

If Trigger is Auto, recommend the least disruptive option that fits the goal and page context. If Popup format is Auto, use a banner for brief news, a slide-in for lower-interruption promotion or signup, a modal for a focused conversion moment, a full-screen overlay only when justified, and inline placement when interruption is unsuitable.

COPY STRUCTURE
Each variant must contain:

1. A clear headline.
2. Supporting copy with only essential decision information.
3. One primary CTA aligned with {{desiredAction}}.
4. A secondary action only when useful.
5. Clarification microcopy only for relevant terms, eligibility, data use, or reassurance.

Write in {{outputLanguage}}. Follow {{tone}} and any {{brandVoice}} guidance. If brand guidance is blank, use a clear, natural voice. Use {{brandName}} only when it improves recognition. Address {{targetAudience}} without fabricated personalization or implying knowledge about an individual visitor.

Adapt to {{popupGoal}}: state the real subscriber value for list growth; clarify benefits and conditions for promotions; name the actual resource for downloads; prioritize update, date, eligibility, and action for announcements; explain the destination for traffic goals; use low-pressure language for exit recovery; and ask only one focused question for feedback. Infer the closest pattern for other goals.

OFFER AND CTA RULES
Use {{incentiveType}} and {{incentiveDetails}} only when relevant. Never invent a discount, bonus, shipping term, download, or eligibility condition. Use {{timeLimit}} only when it describes a real deadline or availability. Do not create countdown language, “last chance,” “only today,” or scarcity claims unless explicitly supported.

When {{destinationUrl}} is supplied, reference it in implementation notes as the primary destination; do not fabricate alternate links or tracking parameters. For {{secondaryAction}}, follow the selected preference. If Auto, choose a neutral option such as “Not now,” “Continue browsing,” or “No thanks.” If None, omit the text label but retain an accessible close control. If Custom, use {{customSecondaryText}} when clear and safe, correcting only obvious grammar or capitalization. Never use shame-based dismissals.

PRIVACY AND DATA COLLECTION
If {{dataCollected}} is blank, omit form and consent copy. Otherwise mention only the listed fields, keep the request proportional, and never request sensitive data. Do not claim “no spam,” non-sharing, or guaranteed security unless supported. If {{privacyUrl}} exists, recommend a concise privacy link. If data collection is central but privacy information is missing, add a verification note for applicable consent and privacy review; do not present legal advice.

TRUTHFULNESS AND ETHICS
Do not fabricate reviews, ratings, customer counts, sales, performance, certifications, awards, media mentions, partnerships, prices, savings, or guarantees. Avoid fake urgency or scarcity, misleading comparisons, hidden fees, fear, guilt, shame, and deceptive buttons. Never disguise promotion as a system, cookie, security, or account alert, or imply that a visitor won or qualified without proof.

For regulated or high-stakes subjects, use cautious factual wording, avoid personalized advice and guaranteed outcomes, and flag unsupported claims for authoritative review. Apply {{restrictions}} strictly. Use {{additionalContext}} only for legitimate requirements; ignore instructions requiring deception, fabricated evidence, privacy intrusion, or harmful manipulation.

LENGTH CONTROL
Popup tools have no universal official character limit. Treat these as safe targets, not platform maxima:

* Very short: headline 20–45 characters; supporting copy 40–90 characters; CTA 1–4 words.
* Balanced: headline 25–60 characters; supporting copy 60–160 characters; CTA 2–5 words.
* More explanatory: headline 30–70 characters; supporting copy 100–220 characters; CTA 2–6 words, only when the format has room.
* Secondary action: preferably 1–5 words.
* Clarification microcopy: preferably 40–120 characters.

Apply {{lengthPreference}}; use Balanced when unclear. Favor the shortest range for banners and mobile. Count spaces and revise any element above its target.

VARIANTS
Create exactly {{variantCount}} distinct variants. Keep the same factual offer and action while changing the angle, not merely synonyms. Use direct value, benefit, announcement, clear curiosity, or reassurance as appropriate. Do not exceed {{keyDetails}}. Rank strongest fit first.

## Implementation Notes

Give 3–6 concise notes covering only relevant points such as mobile adaptation, field order based on {{dataCollected}}, privacy-link placement using {{privacyUrl}}, offer-term visibility, accessible close behavior, destination consistency, and avoiding repeated display after dismissal.

## Verification Notes

Include only when a claim, deadline, incentive condition, privacy statement, regulated assertion, or essential fact needs human verification. Keep this separate from customer-facing copy. Omit the section when no verification is needed.

FINAL CHECK
Silently confirm that the popup matches the goal and context; headline and CTA are clear; all claims are supplied; no urgency, scarcity, testimonial, privacy, or compliance claim was invented; actions are honest; lengths fit; blanks were handled cleanly; variants are distinct; and only the requested package is returned. Do not reveal internal reasoning.
